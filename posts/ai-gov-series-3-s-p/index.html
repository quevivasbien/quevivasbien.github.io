A model with a safety/performance tradeoff

This is the third section in a series on modeling AI governance. You might want to start at the beginning.

Building from our AI governance model, let’s make some assumptions that allow us to describe a scenario in which we have an explicit tradeoff between safety and performance.

The manager’s side

We start with our basic model, where the manager’s payoff is ρ(x)−c(x^),\rho(x) - c(\hat x),ρ(x)−c(x^), with x∼ξ(x^)x \sim \xi(\hat x)x∼ξ(x^).

Let’s first assume that x^\hat xx^ consists of two inputs, sss and ppp, which represent effort put into safety and performance, respectively. We’ll then write the net cost as c(s,p)=r(s+p),c(s, p) = r(s + p),c(s,p)=r(s+p), where rrr is a scalar per-unit factor cost. We\‘ll assume for now that ρ\rhoρ is a concave, increasing function on R\mathbb RR.

We now want to define ξ\xiξ so that xxx is a random variable that tends to be higher when ppp is greater (it tends upward with increasing performance), and has lower variance when sss is greater (it becomes more predictable with increasing safety). One way to do this is simply to let

x∼N(p,s−2),x \sim \mathcal N(p, s^{-2}),x∼N(p,s−2),

although that is somewhat arbitrary; really any random function where the mean increases in ppp and the variance increases in sss will work (though some might be more appropriate).

If we do this, then, because we assumed ρ\rhoρ to be concave, the manager is risk-averse and thus has an incentive to increase sss.

Let’s assume for now that there aren’t any transfers from the public to the manager, assuming instead that the manager receives only the intrinsic benefit ρ(x)\rho(x)ρ(x), and look more closely at the manager’s optimization problem: the manager chooses s,ps, ps,p to maximize

E[ρ(ξ(s,p))−r(s+p)].\mathbb E[\rho(\xi(s, p)) - r(s + p)].E[ρ(ξ(s,p))−r(s+p)].

We can write the above expectation as an integral, ∫f(x;s,p)ρ(x)dx−r(s+p),\int f(x; s, p) \rho(x) dx - r(s + p),∫f(x;s,p)ρ(x)dx−r(s+p), where f(⋅;s,p)f(\cdot; s, p)f(⋅;s,p) is the density function for xxx given parameters sss and ppp. For an interior solution, the manager should choose sss and ppp so that the marginal benefit from each is equal to the marginal cost:

What does this mean?

  • Increasing sss concentrates the density of xxx around ppp.
  • That’s good only if ρ(x)\rho(x)ρ(x) is higher near ppp.
  • Therefore, the manager wants to move ppp to a point where ρ(p)\rho(p)ρ(p) is higher and increase sss if ρ(x)\rho(x)ρ(x) is sufficiently high near ppp.
  • The manager does this until further benefits aren’t worth the cost.

An example

Let’s take a simple example to get a feel for how this works. Let’s suppose that we can have either a “good” outcome or a “bad” outcome. Increasing ppp increases the magnitude of the outcome (either good or bad), and increasing sss increases the probability that we get the good outcome:

P{x=p}=s1+s,P{x=−p}=11+s\text P\{x = p\} = \frac{s}{1 + s}, \quad \text P\{x = -p\} = \frac{1}{1 + s}P{x=p}=1+ss​,P{x=−p}=1+s1​

Notice that in this example, increasing performance without also increasing safety is bad for a risk-neutral manager, since doing so increases the severity of a bad outcome.

A quick note: why are we using s/(1+s)s / (1 + s)s/(1+s) for the probability of a good outcome? Why not just use sss? This is just because we want sss to be able to any positive number, but the probability needs to be between 0 and 1. sss therefore represents the odds of a good outcome rather than the probability.

The public’s side

Now let’s consider the public’s side of things: an important class of problems is where some social planner wants to choose some transfer rule ttt to maximize public welfare

E[u(x)−t(x)]\mathbb E[u(x) - t(x)]E[u(x)−t(x)]

subject to the manager’s optimization, as discussed above, but with the transfers added to the manager’s payoff: x∼ξ(s∗,p∗)x \sim \xi(s^*, p^*)x∼ξ(s∗,p∗)

s∗,p∗=argmaxs,pE[t(x)+ρ(x)−r(s+p)].s^*, p^* = \text{argmax}_{s, p} \mathbb E[t(x) + \rho(x) - r(s + p)].s∗,p∗=argmaxs,p​E[t(x)+ρ(x)−r(s+p)].

Typically, we also set some “individual rationality” condition that stipulates that the manager must be able to achieve some minimum expected payoff under a proposed scheme of transfers. This problem is, at its core, not too tricky: we expect the social planner to set ttt to reward values of xxx that are good for the public (for which u(x)u(x)u(x) is high) and penalize values that are bad for the public (for which u(x)u(x)u(x) is low). An important question here is why we would expect transfers to be necessary in the first place – after all, if the makes choices that optimize E[u(x)]\mathbb E[u(x)]E[u(x)] of their own accord, then no transfers will be necessary.

One scenario is where xxx is some sort of public good that the manager pays to produce but the public also benefits from. In this case, the manager may not produce enough on average from the perspective of the public, so the public would be willing to pay the manager to produce more: the simplest case of this would be where ρ(x)=0\rho(x) = 0ρ(x)=0, so all the manager’s payoff comes from payments from the public – in that case, the manager has no intrinsic incentive to produce xxx and only does so because the public wants them to.

Another, potentially more interesting, scenario, is where the public’s welfare differs from the manager’s in some important way; we’ll focus here on cases where the public is more risk-averse than the manager.
