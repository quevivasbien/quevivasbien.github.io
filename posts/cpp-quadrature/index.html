Numerical integration from scratch in C++

A few days ago I wrote some code, meant to be run on an Arduino, for approximating integrals. This required me to delve deep into nearly-forgotten knowledge about numerical methods; I figured that while I still had this fresh on my mind I’d take the time to share a basic tutorial here.

Setting up the problem

Let’s suppose that we have the integral

∫abf(x)dx.\int_a^b f(x) dx.∫ab​f(x)dx.

The naive approach, which is typically taught in introductory calculus classes, is to pick some step size hhh and divide the interval [a,b][a, b][a,b] into subintervals of size hhh, giving us points

x0=a, x1=a+h, …, xn=xn−1+h, …, xN=bx_0 = a,\ x_1 = a + h,\ \dots,\ x_n = x_{n-1}+h,\ \dots,\ x_N = bx0​=a, x1​=a+h, …, xn​=xn−1​+h, …, xN​=b

Then, we can approximate the integral within each subinterval with something like the midpoint rule,

∫xn−1xnf(x)dx≈hf(xn−1+xn2)\int_{x_{n-1}}^{x_n} f(x) dx \approx hf\left(\frac{x_{n-1} + x_n}{2}\right)∫xn−1​xn​​f(x)dx≈hf(2xn−1​+xn​​)

or the trapezoidal rule,

∫xn−1xnf(x)dx≈h2(f(xn−1)+f(xn)).\int_{x_{n-1}}^{x_n} f(x) dx \approx \frac{h}{2} (f(x_{n-1}) + f(x_n)).∫xn−1​xn​​f(x)dx≈2h​(f(xn−1​)+f(xn​)).

Here’s a simple implementation of the midpoint rule:

// integrate function f on [a, b] with midpoint rule and step size h 
template <typename Function>
double integrate_midpoint(Function&& f, double a, double b, double h) {
    double result = 0;
    double x0 = a;
    while (x0 < b) {
        // set subinterval endpoint
        double x1 = x0 + h;
        if (x1 > b) {
            x1 = b;
            h = b - x0;
        }
        // add midpoint, scaled by h
        result += h * f((x0 + x1) / 2.0);
        // move forward
        x0 = x1;
    }
    return result;
}

Each of these methods will have some amount of error associated with them. That error will be smaller if we use a smaller subinterval size hhh, but it would be nice to be able to be able to bound the error more directly.

How to get arbitrarily low error?

Let’s suppose that we have some level of error, ϵ\epsilonϵ, that we can tolerate. That is, if SnS_nSn​ is our approximation (computed with something like the midpoint or trapezoidal rule), then we want to ensure that

∣∫abf(x)dx−Sn∣<ϵ.\left| \int_a^b f(x) dx - S_n \right| < \epsilon.∣∣​∫ab​f(x)dx−Sn​∣∣​<ϵ.

Here’s one algorithm we could use to come up with a suitable approximation.

  1. Start with an arbitrary step size hhh.
  2. Estimate the error for SnS_nSn​ with a stepsize of hhh
  3. If the error is low enough, accept the approximation SnS_nSn​. Otherwise, choose a smaller hhh, and go back to step 2.

The tricky part here is in step 2: how do we determine the error of a given approximation?

Calculating error for adaptive step sizes

There is one more modification we can make here: depending on how volatile the function is in different parts of its domain, it might make sense to use different step sizes in different parts of the interval [a,b][a, b][a,b]. From here on, we’ll use the notation hnh_nhn​ to denote the size of the nth subinterval (hn=xn−xn−1h_n = x_n - x_{n-1}hn​=xn​−xn−1​), and we’ll call InI_nIn​ the integral over that subinterval:

In=∫xn−1xnf(x)dxI_n = \int_{x_{n-1}}^{x_n} f(x) dxIn​=∫xn−1​xn​​f(x)dx

Let’s see if we can get an estimate for the error in a given subinterval, starting with the trapezoidal rule. We want to figure out the error introduced in the nth block,

In−Tn:=∫xn−1xnf(x)dx−hn2(f(xn−1)+f(xn)).I_n - T_n := \int_{x_{n-1}}^{x_n} f(x) dx - \frac{h_n}{2} (f(x_{n-1}) + f(x_n)).In​−Tn​:=∫xn−1​xn​​f(x)dx−2hn​​(f(xn−1​)+f(xn​)).

The trick is to take the Taylor series approximation of this, which gives us

In−Tn=−hn312f′′(xn−1)+higher-order terms.I_n - T_n = -\frac{h_n^3}{12} f''(x_{n-1}) + \text{higher-order terms}.In​−Tn​=−12hn3​​f′′(xn−1​)+higher-order terms.

This tells us how quickly the error will diminish as we decrease the step size (it scales with hn3h_n^3hn3​), but we still don’t have an exact error estimate, since this depends on the value of f′′(xn−1)f''(x_{n-1})f′′(xn−1​).

However, if we do the same thing with the midpoint approximation:

In−Mn:=∫xn−1xnf(x)dx−hnf(xn−1+xn2),I_n - M_n := \int_{x_{n-1}}^{x_n} f(x) dx - h_n f\left( \frac{x_{n-1} + x_n}{2} \right),In​−Mn​:=∫xn−1​xn​​f(x)dx−hn​f(2xn−1​+xn​​),

we get

In−Mn=hn324f′′(xn−1)+higher-order terms.I_n - M_n = \frac{h_n^3}{24} f''(x_{n-1}) + \text{higher-order terms}.In​−Mn​=24hn3​​f′′(xn−1​)+higher-order terms.

The thing to notice is that

Tn−Mn=(In−Mn)−(In−Tn)≈hn324+hn312=3hn324≈3(In−Mn).T_n - M_n = (I_n - M_n) - (I_n - T_n) \approx \frac{h_n^3}{24} + \frac{h_n^3}{12} = \frac{3h_n^3}{24} \approx 3(I_n - M_n).Tn​−Mn​=(In​−Mn​)−(In​−Tn​)≈24hn3​​+12hn3​​=243hn3​​≈3(In​−Mn​).

This means that we can use the difference between the trapezoidal and midpoint approximations as an estimate for the error of the midpoint approximation! Specifically, 13(Tn−Mn)\frac{1}{3} (T_n - M_n)31​(Tn​−Mn​) is a third-order approximation for the error In−MnI_n - M_nIn​−Mn​.

We can also use this to figure out how big of a step size we should use in order to get a desired amount of error.

The error we have is ϵn≈hn3/24\epsilon_n \approx h_n^3 / 24ϵn​≈hn3​/24, and if we want to get some target error ϵ^n\hat \epsilon_nϵ^n​, we should instead choose h^n\hat h_nh^n​ so that ϵ^n≈h^n3/24\hat \epsilon_n \approx \hat h_n^3 / 24ϵ^n​≈h^n3​/24.

This gives us

ϵ^nϵn≈(h^nhn)3\frac{\hat \epsilon_n}{\epsilon_n} \approx \left( \frac{\hat h_n}{h_n} \right)^3ϵn​ϵ^n​​≈(hn​h^n​​)3

Therefore, we can choose the new step size as

h^n≈hn(ϵ^nϵ)1/3.\hat h_n \approx h_n \left( \frac{\hat \epsilon_n }{\epsilon} \right)^{1/3}.h^n​≈hn​(ϵϵ^n​​)1/3.

This gives us a method for adaptive quadrature with global error ϵ\epsilonϵ:

  1. Start with a step size hnh_nhn​.
  2. Let xn=xn−1+hnx_n = x_{n-1}+h_nxn​=xn−1​+hn​, and compute the triangle rule TnT_nTn​ and midpoint rule MnM_nMn​ approximations on the interval [xn−1,xn][x_{n-1}, x_n][xn−1​,xn​].
  3. Estimate the error of the MnM_nMn​ approximation as ϵn=13(In−Mn)\epsilon_n = \frac{1}{3} (I_n - M_n)ϵn​=31​(In​−Mn​).
  4. The desired error is ϵ^n=h^ϵ/(b−a)\hat \epsilon_n = \hat h \epsilon / (b - a)ϵ^n​=h^ϵ/(b−a). If ϵn≤ϵ^n\epsilon_n \leq \hat \epsilon_nϵn​≤ϵ^n​, then we accept the step size hnh_nhn​ and the approximation MnM_nMn​. Otherwise, we choose a new step size h^n=hn[ϵ^/ϵ]1/3\hat h_n = h_n [\hat\epsilon / \epsilon]^{1/3}h^n​=hn​[ϵ^/ϵ]1/3 and go back to step 2.

Here’s a basic implementation in code:

// integrate function f on [a, b] with midpoint rule and adaptive step size
// error param is desired global error bound
template <typename Function>
double integrate_adaptive(Function&& f, double a, double b, double error) {
    double result = 0;
    double x0 = a;
    double h = b - a;
    while (x0 < b) {
        // set subinterval endpoint
        double x1 = x0 + h;
        if (x1 > b) {
            x1 = b;
            h = b - x0;
        }
        // calculate midpoint and trapezoidal approximations
        double mn = h * f((x0 + x1) / 2.0);
        double tn = h * (f(x0) + f(x1)) / 2.0;
        // compute estimated error of mn, compare to target
        double error_target = error * h / (b - a);
        double error_est = abs(tn - mn) / 3.0;
        if (error_est < error_target) {
            // accept approximation and move on
            result += mn;
            x0 = x1;
        }
        // choose new step size
        h *= 0.9 * pow(error_target / error_est, 1.0 / 3.0);
    }
    return result;
}

There are still a lot of clever things we can do to improve this (e.g., my Arduino code also dynamically adjusts the order of approximation), but for a quick and dirty approach this does a good job.
